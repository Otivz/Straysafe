# STRAYSAFE - COMPLETE SYSTEM PROMPT (PARAGRAPH FORM)

### System Foundation & Navigation
The STRAY-SAFE system is a comprehensive, role-based, AI-assisted stray animal reporting and rescue management platform designed to be implemented with a skeletal, no-color UI using React and Vanilla CSS. The structural foundation relies on a unified top header featuring a logo, a global search bar for incident tracking, a notification center with unread counters for real-time alerts, and a user profile dropdown. This is supported by a persistent sidebar navigation system that dynamically adjusts based on user roles—Citizen, Subdivision Leader, Barangay Staff, and Admin—ensuring a clean and focused workflow across all modules including Dashboard, Reporting, Maps, Analytics, and Audit Logs.

### Citizen Reporting & Interaction
Citizens act as the primary reporting source, utilizing a dashboard to track their submitted incidents and a dedicated reporting modal. This submission process requires the capture of animal images (supported by the `report_media` table), animal type selection, detailed descriptions, and precise GPS/landmark location data. Users can choose between Public and Private visibility for their reports and engage in a community comment section that supports threaded replies (parent-child relationship) for direct communication with subdivision leaders and barangay authorities. Real-time notifications keep citizens updated on their report status as it moves from submission to validation and final resolution.

### Subdivision Leader & AI Validation
Subdivision Leaders serve as the critical validation layer, managing a queue of reports within their assigned subdivision. Each incident view is enhanced by an AI-assisted module that performs image analysis to predict species (Dog/Cat), breed (e.g., Aspin), condition (Injured/Sick/Normal), and behavior tags (Aggressive/Friendly) with specific confidence scores. The system proactively assists leaders by identifying potential duplicate reports through a similarity scoring mechanism (e.g., 85% match based on proximity and visual data). Leaders are responsible for verifying locations and filtering out fake or duplicate entries before escalating legitimate cases to the barangay staff for action.

### Barangay Staff & Rescue Operations
Barangay Staff handle the action and rescue layer, receiving validated requests and managing the full lifecycle of rescue cases. Their workflow includes the generation of official rescue endorsement letters (stored in the `request_letters` table) and the dispatch of response teams via the `case_assignments` module. Staff can update case statuses in real-time—including Picked Up, Under Observation, Impounded, or Released—and record detailed operation logs such as vehicle used, pickup time, and rescue outcomes. All operations are documented with photo proof of pickup and automatic notifications sent to all relevant stakeholders.

### Admin Oversight & Advanced Analytics
Administrators maintain full control over the system, managing user accounts, roles, and subdivision assignments through a robust User Management module. They oversee the entire system's integrity via an Audit Log that tracks every action type, table change, and record ID for security and accountability. The Admin dashboard also features high-level monitoring tools, including hotspot analytics that identify high-density stray animal areas based on incident frequency. This data-driven approach allows for precise resource allocation and performance monitoring across different barangays and subdivisions.

### Admin Heatmap & Density Monitoring
The Admin Heatmap serves as the system's primary "Command Center," offering a map-centric, full-screen interface for real-time stray density monitoring. Utilizing a premium Deep Teal (`#1B4340`) and Brand Orange (`#F97316`) palette, the UI prioritizes visibility through high-z-index glassmorphic overlays that sit above the interactive map container. This includes top-right modular panels for "Monitoring" (offering time-based filters like 24h/7d/30d and category-specific views) and a "Summary" panel that provides ranked hotspot data with risk-level indicators (High/Medium/Low). The interface is anchored by a bottom-positioned "Temporal Activity Flow" chart—a live pulse visualization that maps incident frequency across a 24-hour cycle, allowing administrators to identify peak activity times for more efficient patrol and rescue dispatching.

### Technical Implementation & Data Integrity
The implementation must adhere strictly to a skeletal wireframe aesthetic, utilizing a 1px solid border and 1rem padding system without any background colors, complex gradients, or decorative styling. The frontend is built with React (Vite) and Vanilla CSS, ensuring a responsive Grid/Flexbox layout for both desktop and mobile views. The backend is powered by FastAPI and a MySQL database (straysafe_db), which manages complex relationships between users, reports, media, AI predictions, and status history logs. Every state change in an incident report is recorded in the `status_history` table, ensuring a complete and transparent audit trail from the moment a citizen submits a report to its final resolution.
